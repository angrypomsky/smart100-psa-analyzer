## 초록

열수력(TH) 시뮬레이션 기반 PSA에서 수백 개 시나리오의 출력 파일을 PSA 소프트웨어 입력으로 수작업 변환하는 과정은 분석자에게 상당한 시간 부담과 오류 가능성을 야기한다. 본 논문은 이 변환을 자동화하는 Python 기반 후처리 도구를 제시한다. 이 도구는 MARS-KS TH 출력 파일로부터 원자로 트립(RT) 시점, 피동잔열제거계통(PRHRS) 작동 계통수, 최고 피복재 온도(PCT)를 자동 추출하여, 동적 사건수목(DET) 구조에 대응하는 대화형 Excel 대시보드로 출력한다. 대시보드에서 분석자는 PRHRS 작동 계통수를 기준으로 PCT 분포를 필터링하여 추가 시뮬레이션 없이 성공기준 민감도를 즉시 탐색할 수 있다. 5개 사고유형(LOFW, SBLOCA, GTRN, LSSB, SGTR) 200개씩 총 1,000개 시나리오 데모 데이터셋에서 모든 추출 변수가 물리적 일관성 검증 기준을 만족하였으며, 처리 시간은 일반 PC 기준 30초 미만이다.

## 1. 서론

컴퓨팅 성능은 지속적으로 향상되고 있으며, 이에 따라 PSA 연구에서 열수력 시뮬레이션으로 처리할 수 있는 시나리오 수는 앞으로 더욱 증가할 것으로 전망된다 [5]. 시뮬레이션 처리 속도가 높아질수록 해석해야 할 출력 데이터의 양도 비례하여 늘어나게 된다. 현행 PSA 실무에서 개별 출력 파일의 수동 검토는 열수력 코드 출력을 해석하고 안전계통의 거동을 판단할 수 있는 전문 인력이 직접 수행해야 하는 작업이다. 이는 단순 반복 작업이 아니라 열수력 및 계통 특성에 대한 깊은 이해를 전제로 하며, 시나리오 수에 비례하여 고급 전문 인력의 투입 시간이 선형적으로 증가한다. 시나리오 수가 수백 개 규모에 이를 경우 이 부담은 더욱 심각해지며, 데이터 규모가 커질수록 피로 유발 오류의 확률이 상승하여 시나리오 간 판정 일관성을 확보하기 어려워진다 [5]. 따라서 자동화된 후처리 체계의 필요성은 앞으로 더욱 커질 것이며, 이를 사전에 갖추는 것이 대용량 PSA 환경을 대비하기 위한 핵심 과제가 된다.

Level 1 PSA에서 각 열수력 시뮬레이션 시나리오는 노심 손상 빈도(CDF) 정량화를 위해 안전 기능 성공 여부 판정으로 변환되어야 한다 [1]. 핵심 추출 변수는 원자로 트립(RT) 시점, 안전계통 작동 상태, 최고 피복재 온도(PCT)이며, PCT는 NRC 10 CFR 50.46 기준(1477 K)에 따른 노심 손상 판정의 대리 지표로 활용된다 [2]. 이들 변수를 수백 개의 시나리오에 걸쳐 일관되고 정확하게 추출하는 것이 모든 후속 PSA 분석의 선결 조건이다.

자동화 필요성 외에, 기존 PSA 사건수목의 구조적 한계가 안전 분석의 깊이를 제한한다. 표준 사건수목은 각 안전 기능을 성공 또는 실패의 이진 값으로만 분류한다. 본 연구의 적용 대상인 SMART(System-integrated Modular Advanced ReacTor)는 KAERI가 개발한 100 MWth 일체형 가압경수로로, 4계통의 피동잔열제거계통(PRHRS)을 핵심 안전계통으로 구비하고 있다 [7]. 다중 계통으로 구성된 PRHRS의 경우, 작동 계통수(0~4대)가 사고 결과에 직접적인 영향을 미침에도 이를 이진 표현으로 압축하면 부분적 계통 성능에 관한 정보가 소실된다. 동적 사건수목(DET) 구조는 이산적인 계통수를 분기 변수로 유지함으로써, 계통 가용성의 한계적 변화가 PCT 분포에 미치는 민감도를 이진 사건수목으로는 분석할 수 없는 수준까지 탐색할 수 있게 한다.

본 연구는 이 두 과제를 동시에 해결하는 통합 후처리 도구를 제시한다. 도구는 MARS-KS 출력(Excel)을 자동으로 처리하여 대화형 Excel 대시보드를 출력하며, 분석자는 별도의 시뮬레이션 추가 없이 계통수별 PCT 분포를 즉시 비교할 수 있다. 이는 열수력 시뮬레이션 출력에서 DET 구조 시각화를 거쳐 PSA 소프트웨어 호환 사건수목 자동 생성으로 이어지는 파이프라인의 첫 단계를 구성한다.

## 2. 도구 구성

### 2.1 파이프라인 개요

도구는 Python으로 구현된 단일 스크립트(`psa_analyzer.py`)로 배포되며, 사용자는 GUI 파일 선택 대화창으로 변수명 파일과 시나리오 파일들을 지정한다. 처리가 완료되면 도구는 결과 테이블(CSV)과 대화형 Excel 대시보드를 자동 생성한다. 전체 파이프라인은 Figure 1과 같이 세 단계로 구성된다.

```
[MARS-KS 출력 Excel] → [자동 변수 추출] → [대화형 Excel 대시보드]
      (입력)              (psa_analyzer)         (PSA 분석자 접점)
```

5개 사고유형 모두에 공통으로 적용되는 추출 로직(RT 감지, PCT 계산, PRHRS 판정 알고리즘)은 기반 클래스(`BaseAnalyzer`)에 한 번만 구현한다. 사고유형별 클래스(`LOFWAnalyzer`, `SBLOCAAnalyzer`, `GTRNAnalyzer`, `LSSBAnalyzer`, `SGTRAnalyzer`)는 이 공통 로직을 그대로 재사용하되, 각 사고유형에 맞는 파라미터(Floor, 집계 구간, 대기 시간 등)만 각자 정의한다. 공통 로직의 반복 작성을 피하는 이 구조를 객체지향 프로그래밍에서는 상속(inheritance)이라 부른다. 신규 사고유형 추가 시 파라미터만 지정하면 되므로 코드 오류 가능성을 낮추고 유지보수성을 높인다.

### 2.2 입출력 명세

**입력:**
- MARS-KS 출력 Excel 파일 (시나리오당 1개, 다중 선택 가능)
- 변수명 파일 (노형별 컬럼 매핑 기준)

**출력:**
- 시나리오별 결과 테이블 (CSV): `Scenario`, `Reactor_Trip`, `PRHRS_count`, `PCT_max`, `PCT_time`, `Outcome`
- 사고유형별 대화형 대시보드 (Excel): 계통수 필터별 PCT 분포 차트, 시나리오 목록, 요약 통계

### 2.3 자동 변수 추출

#### 변수명 자동 매핑 (VarMapper)

MARS-KS 변수명 형식은 노형에 따라 상이하다. VarMapper는 변수명 파일을 파싱하여 키워드 매칭으로 컬럼을 기능별 역할에 자동 대응시킨다(PRHRS HX 열출력, 피복재 온도, 원자로 출력). 노형 변경 시 키워드 사전만 수정하면 코드 변경 없이 적용된다.

#### 원자로 트립 감지 및 PCT 추출 구간 설정

원자로 출력(rktpow)이 초기값의 20% 이하로 감소하는 첫 시점을 RT 시점으로 판정한다. PCT 추출 구간 시작에는 더 엄격한 10% 기준을 적용한다. 두 임계값을 달리 설정하는 이유는 역할의 차이에 있다. 20% 기준은 트립 신호 발생 여부를 판정하는 데 사용되며, 트립 신호가 발생한 직후에도 관성으로 인한 잔류 출력이 수십 초간 지속된다. PCT는 이 잔류 출력 과도구간이 완전히 종료된 이후 냉각 단계에서 추출해야 실제 최고 피복재 온도를 정확히 포착할 수 있으므로, 더 엄격한 10% 기준으로 PCT 추출 구간 시작점을 정의한다. MARS-KS Excel 출력 첫 행의 아티팩트(rktpow=0) 문제를 회피하기 위해 초기 출력값은 첫 번째 양수 rktpow 값으로 산정한다.

#### PRHRS 작동 계통수 판정

각 PRHRS HX *i*의 RT 이후 대표 열출력 $Q_i$를 산출하고, 아래 상대비교 기준으로 작동 여부를 판정한다.

$$Q_i \geq Q_{\max} \times R_{\text{기준}}$$

$Q_{\max}$는 4개 계통 중 최대값, $R_{\text{기준}}$은 기본 10%이다. 고정 절대 임계값 방식과 달리 이 상대 기준은 시나리오별 열부하 변화에 자동 적응한다. 사고유형별 파라미터는 Table 1에 정리하였다.

**Table 1. 사고유형별 PRHRS 알고리즘 파라미터**

| 파라미터 | LOFW | SBLOCA | GTRN | LSSB | SGTR |
|---------|------|--------|------|------|------|
| 집계 구간 | 전체 평균 | 초기 3h | 전체 평균 | 전체 평균 | 전체 평균 |
| 대기 시간 (s) | 100 | 100 | 100 | 100 | 100 |
| Floor (W) | 1×10⁶ | 1×10⁶ | 1×10⁶ | 1×10⁶ | 1×10⁶ |
| 상대비교 기준 | 10% | 10% | 10% | 10% | 10% |

Floor = 1×10⁶ W는 모든 사고유형에 균일하게 적용된다. 실제 데이터 분석에서 작동 계통의 RT 이후 PRHRS HX 출력은 최소 수 MW 수준인 반면, 비작동 계통(열교환기 미연결 등)의 잔류 출력은 수백 kW 이하로 1×10⁶ W 기준에서 명확히 분리됨을 확인하였다.

SBLOCA의 경우 RT 이후 PRHRS 열출력이 초기 수 MW에서 시작하여 붕괴열 감소에 따라 자연 감소한다. 이는 계통 고장이 아닌 열원(붕괴열)의 감소에 따른 정상적인 거동이다. 실제 데이터에서 4계통 작동 케이스는 RT+약 11,350~11,600s에 1×10⁶ W 이하로 감소한다. 전체 시간 평균을 사용하면 이 감소 구간의 낮은 출력값이 반영되어 계통수가 과소 평가될 수 있으므로, RT+대기 이후 10,800s(3h) 초기 창을 집계 구간으로 제한한다. 이 창 내에서 1~4계통 작동 케이스는 모두 수 MW 수준으로 안정 작동하여 상대비교 기준이 유효하다. 0계통 케이스(피크 출력 ~2 kW)는 Floor에서 완전히 분리된다.

SGTR은 RT 이후 PRHRS가 수십 시간 연속 작동하나 후반부에 열출력이 감소한다. 후반 30% 집계를 사용하면 이 감소 구간이 집계 구간에 포함되어 작동 계통이 0계통으로 오분류될 수 있으므로, 전체 평균을 사용한다. LSSB는 파단 초기 대형 과도현상 이후 작동/비작동 계통 간 출력 비율이 3~5% 이하로 극단적으로 분리되므로, 10% 상대 기준으로도 정확한 분류가 가능함을 데이터 검증으로 확인하였다.

#### PCT 추출 및 CD 판정

10% 기준 RT 시점 이후 최대 피복재 온도를 PCT로 추출한다. 10,000 K 초과 비물리적 이상값은 제거한다. NRC 10 CFR 50.46 기준[2]: PCT ≥ 1477 K → CD, PCT < 1477 K → OK.

### 2.4 대화형 Excel 대시보드

대시보드는 PSA 분석자가 추가 코딩 없이 직접 사용하는 도구 접점이다. 각 사고유형별 독립 시트로 구성되며, 주요 기능은 다음과 같다.

| 기능 | 내용 |
|------|------|
| PRHRS 계통수 필터 | 드롭다운으로 계통수(0~4) 선택 시 PCT 분포 차트 즉시 갱신 |
| PCT 분포 시각화 | 선택된 계통수 조건의 히스토그램 및 통계 요약 (min, mean, max, P95) |
| 시나리오 목록 | 필터 조건에 해당하는 시나리오 파일명, RT 결과, PCT, CD 판정 표시 |
| 전체 분포 비교 | 계통수별 PCT 분포를 중첩하여 단조성 확인 |

이 구조는 DET와 동등하다. 이진 성공/실패 분기 대신 PRHRS 계통수(0~4)를 이산 분기 변수로 정의하고 각 상태의 PCT 분포를 직접 시각화함으로써, 성공기준 변경(예: "3계통 이상"에서 "2계통 이상"으로 완화) 시 노심 손상 빈도에 미치는 영향을 추가 시뮬레이션 없이 즉시 확인할 수 있다.

## 3. 데모 결과

### 3.1 검증 데이터셋 및 알고리즘 성능

알고리즘 검증에는 실제 MARS-KS 시뮬레이션 결과를 사용하였다. 5개 사고유형 전체에 대해 RT 감지 및 PRHRS 계통수 판정 결과를 수작업 검토값과 비교하였으며, 두 알고리즘 모두 100% 일치를 달성하였다. 실제 시뮬레이션 시나리오는 설계 조건 범위 내에서 수행되어 전 시나리오가 CD 미발생(OK)이며, 추출된 PCT 분포는 700~1,400 K 범위이다. CD 임계값(1477 K) 이상 시나리오에 대한 알고리즘 검증은 향후 추가 시뮬레이션 수행 후 진행할 예정이다.

### 3.2 대시보드 데모 데이터셋

실제 시뮬레이션 데이터는 전 시나리오가 OK이므로 대시보드의 CD 필터링 기능을 시연하기에 적합하지 않다. 이를 위해 5개 사고유형별 200개씩 총 1,000개 시나리오의 합성 데모 데이터셋을 별도로 생성하였다. 층화 표본추출로 PRHRS 계통수 수준별·RT 결과별 비례 분포를 구성하였으며, CD 시나리오도 포함하였다. 아래 4개 물리적 일관성 규칙으로 데이터셋 품질을 검증하였다.

- **규칙 A**: PCT–결과 일관성 (1477 K 임계값)
- **규칙 B**: PRHRS 단조성 (계통수 증가 → PCT 감소)
- **규칙 C**: ATWS 패널티 (RT 실패 → PCT 상승)
- **규칙 D**: Feed-and-Bleed 작동 조건 제약

## 4. 결론

본 연구는 MARS-KS TH 시뮬레이션 출력을 PSA 분석에 직접 활용 가능한 대화형 Excel 대시보드로 자동 변환하는 도구를 제시하였다. 도구는 단일 Python 스크립트로 배포되며, GUI 기반 파일 선택만으로 RT 감지, PRHRS 작동 계통수 판정, PCT 추출, CD 판정 및 대시보드 생성을 일괄 수행한다. 대시보드는 PRHRS 작동 계통수를 이산 분기 변수로 정의하는 DET 구조를 채택하여, 분석자가 성공기준 민감도를 추가 시뮬레이션 없이 탐색할 수 있는 환경을 제공한다.

상대비교 기반 PRHRS 계통수 판정 알고리즘은 초기 조건 변화에 강인한 분류를 제공하며, VarMapper 모듈은 노형 변경 시에도 코드 수정 없이 재적용 가능한 확장성을 확보한다. 1,000개 데모 시나리오에서 모든 추출 변수의 물리적 일관성이 확인되었고 처리 시간은 30초 미만이다.

향후 과제로는 실제 MARS-KS 시뮬레이션 결과 적용, ADS·PSIS·SIT 등 추가 안전계통 확장, AIMS-PSA 호환 사건수목 형식(.ket) 직접 출력 구현이 있다. 완성된 파이프라인은 TH 시뮬레이션 데이터로부터 PSA 소프트웨어 입력 파일을 완전 자동 생성하는 통합 후처리 체계를 목표로 한다.
