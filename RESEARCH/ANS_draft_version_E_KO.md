## 1. 서론

원자력발전소의 안전성을 정량적으로 평가하기 위해 확률론적 위험도 평가(Probabilistic Risk Assessment, PRA)는 필수적인 도구로 자리잡았다 [1]. PRA에서 각 사고 시나리오는 사건수목(ET)을 통해 표현되며, 노심 손상 빈도(CDF)를 정량화하기 위해서는 각 시나리오가 안전 기능의 성공 또는 실패 여부로 변환되어야 한다. 그러나 표준 ET는 각 안전 기능을 성공/실패의 이진 값으로만 분류하는 구조적 한계를 가지며, 이로 인해 다중 계통으로 구성되거나 시간 의존적 거동을 보이는 안전계통에 대한 세분화된 시나리오 검토에는 한계가 있다. 본 연구의 검증 대상인 SMART(System-integrated Modular Advanced ReacTor)는 KAERI가 개발한 100 MWth 일체형 가압경수로로, 4계통의 피동잔열제거계통(PRHRS)을 핵심 안전계통으로 구비하고 있다 [7]. 이러한 다중 계통의 경우 작동수(0~4대)가 사고 결과에 직접적인 영향을 미침에도, 이를 이진 표현으로 압축하면 부분적 계통 성능에 관한 정보가 소실된다.

이러한 한계를 극복하기 위해 동적 사건수목(Dynamic Event Tree, DET) 방법론이 새로운 접근법으로 부상하고 있다. DET는 이산적인 안전계통 작동수를 분기 변수로 유지함으로써, 계통 가용성의 한계적 변화가 최고 피복재 온도(PCT) 분포에 미치는 민감도를 표준 ET로는 분석할 수 없는 수준까지 탐색할 수 있게 한다. 그러나 DET는 표준 ET에 비해 분기되는 가지 수가 훨씬 많고, 각 분기 조건에 대응하는 결정론적 열수력(TH) 시뮬레이션 결과가 뒷받침되어야 하므로, 안전계통 작동수 조합이 늘어날수록 필요한 시뮬레이션 데이터의 양도 기하급수적으로 증가한다.

문제는 이렇게 생성된 방대한 양의 TH 시뮬레이션 결과를 다시 PRA로 가져와 사건수목으로 정리하는 작업이다. 시나리오 수가 수백 개 규모에 이르면, 개별 출력 파일을 검토하여 원자로 트립(RT) 시점, 안전계통 작동 상태, PCT(NRC 10 CFR 50.46 기준 1477 K에 따른 노심 손상 판정의 대리 지표 [2])를 추출하고 이를 일관되게 PRA 입력으로 변환하는 작업은, 열수력 코드 출력을 해석하고 안전계통의 거동을 판단할 수 있는 전문 인력이 직접 수행해야 하는 작업이다. 이는 단순 반복 작업이 아니라 열수력 및 계통 특성에 대한 깊은 이해를 전제로 하며, 시나리오 수에 비례하여 고급 전문 인력의 투입 시간이 선형적으로 증가한다. 데이터 규모가 커질수록 피로 유발 오류의 확률도 상승하여 시나리오 간 판정 일관성을 확보하기 어려워진다 [5]. 결과적으로 DET 도입으로 시뮬레이션 규모가 커질수록 전문 인력의 투입 시간, 즉 인적비용(human cost)이 함께 증가하여 DET 결과를 PRA로 가져오는 데 드는 비용이 그 활용을 가로막는 비용 장벽이 된다.

따라서 DET가 생성하는 대량의 TH 시뮬레이션 결과를 자동으로 처리하여 PRA에서 즉시 활용 가능한 형태로 변환하는 후처리 기능이 필요하다. 본 연구는 이러한 배경에서 통합 후처리 도구를 제시한다. 도구는 MARS-KS 출력(Excel)을 자동으로 처리하여 대화형 Excel 대시보드를 출력하며, 분석자는 별도의 시뮬레이션 추가 없이 안전계통 작동수별 PCT 분포를 즉시 비교할 수 있다. 이는 열수력 시뮬레이션 출력에서 DET 구조 시각화를 거쳐 PRA 소프트웨어 호환 사건수목 자동 생성으로 이어지는 파이프라인의 첫 단계를 구성한다.

## 2. 파이프라인

도구는 Python으로 구현된 단일 스크립트(`psa_analyzer.py`)로 배포되며, 사용자는 GUI 파일 선택 대화창으로 변수명 파일과 시나리오 파일들을 지정한다. 처리가 완료되면 도구는 결과 테이블(CSV)과 대화형 Excel 대시보드를 자동 생성한다. 전체 파이프라인은 Figure 1과 같이 세 단계로 구성된다.

```
[MARS-KS 출력 Excel] → [자동 변수 추출] → [대화형 Excel 대시보드]
      (입력)              (psa_analyzer)         (PRA 분석자 접점)
```

5개 사고유형 모두에 공통으로 적용되는 추출 로직(RT 감지, PCT 계산, 안전계통 작동 판정 알고리즘)은 기반 클래스(`BaseAnalyzer`)에 한 번만 구현한다. 사고유형별 클래스(`LOFWAnalyzer`, `SBLOCAAnalyzer`, `GTRNAnalyzer`, `LSSBAnalyzer`, `SGTRAnalyzer`)는 이 공통 로직을 그대로 재사용하되, 각 사고유형에 맞는 파라미터(Floor, 집계 구간, 대기 시간 등)만 각자 정의한다. 공통 로직의 반복 작성을 피하는 이 구조를 객체지향 프로그래밍에서는 상속(inheritance)이라 부른다. 신규 사고유형 추가 시 파라미터만 지정하면 되므로 코드 오류 가능성을 낮추고 유지보수성을 높인다.

도구의 입력은 MARS-KS 출력 Excel 파일과 노형별 컬럼 매핑 기준이 되는 변수명 파일이다. 시나리오 1개당 파일 1개를 기본으로 하되, 여러 시나리오 파일을 한 번에 다중 선택하여 일괄 처리할 수 있다. 처리 결과로는 시나리오별 결과 테이블(CSV: `Scenario`, `Reactor_Trip`, `Safety_System_Count`, `PCT_max`, `PCT_time`, `Outcome`)과, 사고유형별 안전계통 작동수 필터별 PCT 분포 차트·시나리오 목록·요약 통계를 포함하는 대화형 대시보드(Excel)가 생성된다.

변수 추출 과정에서는 먼저 VarMapper 모듈이 변수명 파일을 파싱하여 키워드 매칭으로 컬럼을 기능별 역할(안전계통 출력, 피복재 온도, 원자로 출력)에 자동 대응시킨다. MARS-KS 변수명 형식은 노형에 따라 상이하지만, 노형 변경 시에는 키워드 사전만 수정하면 코드 변경 없이 적용할 수 있다.

원자로 트립(RT) 시점은 원자로 출력(rktpow)이 초기값의 20% 이하로 감소하는 첫 시점으로 판정한다. 반면 PCT 추출 구간의 시작에는 더 엄격한 10% 기준을 적용하는데, 이는 두 임계값의 역할이 다르기 때문이다. 20% 기준은 트립 신호 발생 여부를 판정하는 데 사용되며, 트립 신호가 발생한 직후에도 관성으로 인한 잔류 출력이 수십 초간 지속된다. PCT는 이 잔류 출력 과도구간이 완전히 종료된 이후 냉각 단계에서 추출해야 실제 최고 피복재 온도를 정확히 포착할 수 있으므로, 더 엄격한 10% 기준으로 PCT 추출 구간 시작점을 정의한다. 또한 MARS-KS Excel 출력 첫 행의 아티팩트(rktpow=0) 문제를 회피하기 위해 초기 출력값은 첫 번째 양수 rktpow 값으로 산정한다.

안전계통 작동수 판정에서는 각 안전계통 *i*의 RT 이후 대표 출력 $Q_i$를 산출한다. 전체 계통 중 최대 출력 $Q_{\max}$를 100으로 정규화했을 때, $Q_i$가 그 10% 미만이면 해당 계통은 작동하지 않은 것으로 판정한다. 즉,

$$Q_i \geq Q_{\max} \times R_{\text{기준}}$$

을 만족하는 계통만 작동 중으로 집계하며, $R_{\text{기준}}$은 기본 10%이다. 고정 절대 임계값 방식과 달리 이 상대 기준은 시나리오별 열부하 변화에 자동 적응한다. 사고유형별 파라미터는 Table 1에 정리하였다.

**Table 1. 사고유형별 안전계통 작동 판정 알고리즘 파라미터 (모든 파라미터는 RT 이후 구간에 적용)**

| 파라미터 | LOFW | SBLOCA | GTRN | LSSB | SGTR |
|---------|------|--------|------|------|------|
| 집계 구간 (RT 이후) | 전체 평균 | 초기 3h | 전체 평균 | 전체 평균 | 전체 평균 |
| 대기 시간 (RT 이후, s) | 100 | 100 | 100 | 100 | 100 |
| Floor (W) | 1×10⁶ | 1×10⁶ | 1×10⁶ | 1×10⁶ | 1×10⁶ |
| 상대비교 기준 | 10% | 10% | 10% | 10% | 10% |

> **주의:** Floor 값(현재 1×10⁶ W)은 노형 및 열출력 규모에 따라 달라질 수 있으므로, 다른 노형에 적용할 경우 사용자가 직접 조정해야 한다. 대기 시간, 집계 구간, 상대비교 기준 역시 노형별 과도 특성을 반영하여 검토가 필요하다.

Floor = 1×10⁶ W는 모든 사고유형에 균일하게 적용된다. 실제 데이터 분석에서 작동 계통의 RT 이후 출력은 최소 수 MW 수준인 반면, 비작동 계통(열교환기 미연결 등)의 잔류 출력은 수백 kW 이하로 1×10⁶ W 기준에서 명확히 분리됨을 확인하였다.

SBLOCA의 경우 RT 이후 안전계통 출력이 초기 수 MW에서 시작하여 붕괴열 감소에 따라 자연 감소한다. 이는 계통 고장이 아닌 열원(붕괴열)의 감소에 따른 정상적인 거동이다. 실제 데이터에서 4계통 작동 케이스는 RT+약 11,350~11,600s에 1×10⁶ W 이하로 감소한다. 전체 시간 평균을 사용하면 이 감소 구간의 낮은 출력값이 반영되어 계통수가 과소 평가될 수 있으므로, RT+대기 이후 10,800s(3h) 초기 창을 집계 구간으로 제한한다. 이 창 내에서 1~4계통 작동 케이스는 모두 수 MW 수준으로 안정 작동하여 상대비교 기준이 유효하다. 0계통 케이스(피크 출력 ~2 kW)는 Floor에서 완전히 분리된다.

PCT 추출 및 CD 판정은 위에서 정의한 10% 기준 RT 시점 이후 최대 피복재 온도를 PCT로 추출하는 방식으로 이루어진다. 10,000 K 초과 비물리적 이상값은 제거한다. NRC 10 CFR 50.46 기준[2]: PCT ≥ 1477 K → CD, PCT < 1477 K → OK.

## 3. Excel 대시보드

대화형 Excel 대시보드는 본 도구의 핵심 출력으로, PRA 분석자가 추가 코딩 없이 직접 사용하는 도구 접점이다. 각 사고유형별 독립 시트로 구성되며, 주요 기능은 다음과 같다.

| 기능 | 내용 |
|------|------|
| 안전계통 작동수 필터 | 드롭다운으로 작동수(0~4) 선택 시 PCT 분포 차트 즉시 갱신 |
| PCT 분포 시각화 | 선택된 작동수 조건의 히스토그램 및 통계 요약 (min, mean, max, P95) |
| 시나리오 목록 | 필터 조건에 해당하는 시나리오 파일명, RT 결과, PCT, CD 판정 표시 |
| 전체 분포 비교 | 작동수별 PCT 분포를 중첩하여 단조성 확인 |

이 구조는 DET와 동등하다. 이진 성공/실패 분기 대신 안전계통 작동수(0~4)를 이산 분기 변수로 정의하고 각 상태의 PCT 분포를 직접 시각화함으로써, 성공기준 변경(예: "3계통 이상"에서 "2계통 이상"으로 완화) 시 노심 손상 빈도에 미치는 영향을 추가 시뮬레이션 없이 즉시 확인할 수 있다. 분석자는 Excel 환경에서 친숙한 드롭다운 인터페이스만으로 수백 개 시나리오의 통계를 실시간으로 탐색하며, 별도의 소프트웨어 설치나 코딩 지식 없이 성공기준 민감도 분석을 수행할 수 있다.

## 4. 결과

알고리즘 검증에는 실제 MARS-KS 시뮬레이션 결과를 사용하였다. 5개 사고유형 전체에 대해 RT 감지 및 안전계통 작동수 판정 결과를 수작업 검토값과 비교하였으며, 두 알고리즘 모두 100% 일치를 달성하였다. 다만, 현재 확보된 시뮬레이션 데이터는 설계 기준 사고 범위 내의 제한된 규모로, PCT 분포 상단(고온 구간)에서의 안전계통 거동은 충분히 포착되지 않았다. 특히 초기 안전계통 출력이 지배적인 구간 이후 열수력 거동의 다양성이 데이터에 충분히 반영되지 않아, 이 구간에 대한 알고리즘 검증은 향후 대규모 시뮬레이션 수행 후 진행할 예정이다.

실제 시뮬레이션 데이터는 확보 규모가 제한적이고 안전계통 출력이 지배적인 구간에 집중되어 있어, 대시보드의 전 구간 PCT 분포 필터링 기능을 충분히 시연하기에 적합하지 않다. 이를 위해 5개 사고유형별 200개씩 총 1,000개 시나리오의 합성 데모 데이터셋을 별도로 생성하였다. 안전계통 작동수 수준별·RT 결과별로 균형 잡힌 분포를 구성하였으며, CD 시나리오도 포함하였다. 데이터셋 품질은 네 가지 물리적 일관성 규칙, 즉 PCT–결과 일관성(1477 K 임계값), 안전계통 단조성(작동수 증가 시 PCT 감소), ATWS 패널티(RT 실패 시 PCT 상승), Feed-and-Bleed 작동 조건 제약으로 검증하였으며, 1,000개 전 시나리오에서 규칙 위반은 0건이었다.

**Table 2. 도구 성능 요약**

| 항목 | 검증 대상 | 결과 |
|------|---------|------|
| RT 감지 | 실제 MARS-KS 데이터, 수작업 비교, 5개 사고유형 | 100% 일치 |

## 5. 결론

본 연구는 MARS-KS TH 시뮬레이션 출력을 PRA 분석에 직접 활용 가능한 대화형 Excel 대시보드로 자동 변환하는 도구를 제시하였다. 도구는 단일 Python 스크립트로 배포되며, GUI 기반 파일 선택만으로 RT 감지, 안전계통 작동수 판정, PCT 추출, CD 판정 및 대시보드 생성을 일괄 수행한다. 도구의 핵심 기여는 대화형 Excel 대시보드로, 안전계통 작동수를 이산 분기 변수로 정의하는 DET 구조를 채택하여 분석자가 추가 시뮬레이션 없이 성공기준 민감도를 즉시 탐색할 수 있는 환경을 제공한다. Excel 친화적 인터페이스 덕분에 코딩 지식 없이도 수백 개 시나리오의 PCT 분포를 실시간으로 비교·분석할 수 있다.

상대비교 기반 안전계통 작동수 판정 알고리즘은 초기 조건 변화에 강인한 분류를 제공하며, VarMapper 모듈은 노형 변경 시에도 코드 수정 없이 재적용 가능한 확장성을 확보한다. 1,000개 데모 시나리오에서 모든 추출 변수의 물리적 일관성이 확인되었고 처리 시간은 30초 미만이다.

향후 과제로는 실제 대량의 MARS-KS 시뮬레이션 결과 적용을 통한 알고리즘 전 구간 검증, ADS·PSIS·SIT 등 추가 안전계통 확장, 도구 출력 결과를 AIMS-PSA 호환 사건수목(ET) 파일 형식(.ket)으로 직접 출력하는 기능 구현이 있다. 완성된 파이프라인의 최종 목표는 대규모 열수력 시뮬레이션 데이터를 후처리하여 .ket 파일로 생성하는 것으로, 분석자가 열수력 데이터로부터 사건수목을 직접 작도할 필요 없이 AIMS-PSA에서 해당 .ket 파일을 열어 완성된 사건수목 구조를 즉시 확인할 수 있도록 하는 통합 후처리 체계 구축이다.
