요약
코드 출력을 대상으로 RT, prhrs, pct를 얘기하는데 이거를 이렇게 명시하는게 나은지? (몇 개만 나열하면 그것만 한것 같은 느낌이 나서)

서론
prhrs를 핵심으로 하는 자동 후처리 프레임워크 > prhrs를 강조하는게 나은지?

방법론
객체지향 설계 용어 설명 필요
사고유형도 5개라고 하는데 굳이 언급할 필요가 있나 싶음
풀백 < 용어 설명필요
집게 열출력 용어 설명
table1 대기시간 lofw만 다른 이유?
계통수 보정 다 0으로 없으면 보정이라는 단어 지워도 될거 같음
집계구간을 다르게 설정한게 설명되어있긴한데 왜 저런 알고리즘으로 짰는지에 대한 근거가 있어야 할거 같음
이진 사건수목 출력은 아직 구현 단계는 아님. 추후 발전으로 돌려야 할듯

검증 결과
층화 표본추출 용어 설명
규칙 a,b,c,d가 어떻게 작동하는지에 대한 설명 필요 (나도 공부 필요)
smr의 현실적인 확률분포를 반영한다 > 확인 필요(참고문헌)
.ket파일 연계 x 추후 업데이트로 빼야함

결론
내 생각 - 대량 열수력 데이터 후처리를 통한 et 자동화의 첫걸음 / 데이터 후처리 > det 구조로 확인 > et구조로 변환이라는 et 자동추출이라는 큰 틀에서의 첫 단계라는걸 말하고 싶음
따라서 주요 기여로 이거의 첫 걸음을 했다. 나머지 4개는 괜찮은듯. 다 확장 가능하니까.




그림 후보
4페이지 제한 + 표 2개 기준으로 그림 최대 2~3개 가능

★★★ 필수

그림 1 - 절대 임계값 vs 상대 비교 알고리즘 비교 예시
같은 시나리오를 두 방식으로 판정했을 때 결과가 달라지는 케이스 시각화
X축=시간, Y축=HX 열출력, 절대 임계선과 상대 기준선을 겹쳐 표시
논문 핵심 기여(왜 상대 비교가 필요한가)를 한눈에 보여줌
데모 데이터에서 바로 생성 가능

그림 2 - PCT 민감도 분석 (CD 임계값 1300~1600 K 구간)
X축=PCT 임계값, Y축=CD 판정 건수(또는 비율)
사고유형별 5개 선 + NRC 1477 K 기준선 표시
알고리즘 적용 가능성을 정량적으로 보여주는 유일한 결과 그림
데모 데이터에서 바로 생성 가능

★★ 권장 (지면 여유 시)

그림 3 - 프레임워크 전체 파이프라인 흐름도
입력(MARS-KS Excel) → VarMapper → RT 감지 → PRHRS 판정 → PCT 추출 → 출력 순서를 박스 다이어그램으로
전체 구조 파악에 도움되나 본문 설명만으로도 전달 가능 → 지면 부족 시 생략

제외 후보 및 이유
클래스 상속 구조도 - 소프트웨어 구현 세부사항, ANS 청중 불필요
PRHRS 알고리즘 5단계 흐름도 - Table I + 본문으로 충분
PCT 분포 히스토그램 - 데모 데이터임을 강조하게 되어 오히려 약점 부각
RT 감지 rktpow 곡선 - 80% 감소 기준은 직관적, 그림 없이 이해 가능
