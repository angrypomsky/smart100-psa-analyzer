## 2. Abstract

### 한국어

확률론적 안전성 평가(PSA) 수행 시 다수의 열수력 시뮬레이션 결과를 처리하는
후처리 과정은 반복적이고 수작업 의존도가 높아 인적 오류 및 판정 비일관성의
위험이 존재한다. 본 연구에서는 MARS-KS 열수력 해석 코드 출력 결과를 대상으로
원자로 트립(RT) 감지, 피동잔열제거계통(PRHRS) 작동 계통수 자동 판정,
최고 피복재 온도(PCT) 추출을 수행하는 자동 후처리 알고리즘 프레임워크를
개발하였다. 핵심 기여는 기존 절대 임계값 방식을 대체하는 상대 비교 기반
PRHRS 판정 알고리즘으로, 각 계통의 열출력 값이 최대값의 일정 비율 이상인
경우 작동으로 판정함으로써 시뮬레이션 초기 조건 변화에 강인한 판정이 가능하다.
또한 변수명 파일 기반 자동 컬럼 매핑(VarMapper) 기법을 통해 노형 변경 시에도
코드 수정 없이 대응할 수 있는 확장성을 확보하였다. 개발된 프레임워크는
급수상실사고(LOFW), 소형냉각재상실사고(SBLOCA), 일반과도(GTRN),
대형 2차측 파단(LSSB), 증기발생기 전열관 파단(SGTR)의 5개 사고유형에
적용되었으며, 사고유형별 물리적 특성에 근거한 파라미터 조정 기준을 제시하였다.
NRC 10 CFR 50.46 기준(1477 K) 대비 CD 임계값 민감도 분석을 통해
알고리즘의 적용 가능성을 검증하였다.

**핵심어**: 확률론적 안전성 평가, 피동잔열제거계통, 원자로 트립 감지,
후처리 자동화, MARS-KS

---

### English

In probabilistic safety assessment (PSA), the post-processing of large-scale
thermal-hydraulic simulation results is highly repetitive and manually intensive,
posing risks of human error and inconsistent failure judgments. This study presents
an automated post-processing algorithm framework for MARS-KS thermal-hydraulic
code outputs, capable of automatically detecting reactor trip (RT) timing,
determining the number of operating passive residual heat removal system (PRHRS)
trains, and extracting peak cladding temperature (PCT). The primary contribution
is a relative-comparison-based PRHRS assessment algorithm that replaces
conventional absolute threshold methods; a train is classified as active when
its heat output exceeds a defined fraction of the maximum observed value,
ensuring robustness against variations in initial simulation conditions. A
variable-name-file-based automatic column mapping scheme (VarMapper) is also
introduced to enable applicability across different reactor designs without code
modification. The framework is applied to five accident categories — loss of
feedwater (LOFW), small-break LOCA (SBLOCA), general transient (GTRN),
large secondary-side break (LSSB), and steam generator tube rupture (SGTR) —
with physically justified parameter configurations for each. Applicability is
validated through core damage threshold sensitivity analysis relative to the
NRC 10 CFR 50.46 criterion (1477 K).

**Keywords**: Probabilistic Safety Assessment, Passive Residual Heat Removal
System, Reactor Trip Detection, Post-processing Automation, MARS-KS

## 3. 논문 구성 (섹션별 주요 내용)

### 3.1 서론 (Introduction)

원자력 발전소의 안전성을 정량적으로 평가하기 위한 확률론적 안전성 평가(Probabilistic Safety Assessment, PSA)는 IAEA 및 NRC의 규제 체계에서 핵심적인 역할을 수행한다 [1]. Level 1 PSA는 노심 손상 빈도(Core Damage Frequency, CDF)를 정량화하는 과정으로, 이를 위해 다수의 사고 시나리오에 대한 열수력 시뮬레이션이 수행된다. 각 시나리오의 결과는 원자로 트립(Reactor Trip, RT) 성공 여부, 피동잔열제거계통(Passive Residual Heat Removal System, PRHRS) 작동 계통수, 최고 피복재 온도(Peak Cladding Temperature, PCT)와 같은 핵심 변수를 기반으로 안전 기능 성공 여부가 판정된다 [1][2].

최근 최적추정치+불확실도(Best Estimate Plus Uncertainty, BEPU) 방법론의 발전과 함께, PSA 수행 시 수십 개에서 수백 개에 달하는 대용량 시나리오 샘플을 생성하여 성공기준의 통계적 불확실도를 정량화하는 접근이 주목받고 있다 [3][5]. 이러한 대용량 PSA 환경에서는 RT 판정 시점, PRHRS 작동 여부 판단 기준, PCT 임계값 등 성공기준 파라미터의 민감도를 체계적으로 평가하는 것이 중요한 과제로 떠오르고 있다. 그러나 이를 위해서는 대규모 시뮬레이션 결과를 일관된 기준으로 신속하게 처리할 수 있는 자동화된 후처리 체계가 선결 조건으로 요구된다.

현행 PSA 실무에서 열수력 시뮬레이션 결과의 후처리는 분석자가 출력 파일을 수동으로 검토하여 각 변수를 추출하고 성공 기준을 적용하는 방식에 의존하는 경우가 많다. 수작업 기반 후처리는 분석자 간 판정 기준의 일관성을 보장하기 어려우며, 특히 PRHRS와 같은 피동 안전계통의 작동 여부는 계통별 열출력의 시간적 거동을 종합적으로 고려해야 하므로 단순한 임계값 비교만으로는 정확한 판정이 어렵다 [9][10]. 이에 따라 열수력 코드 출력을 자동으로 처리하고 사고유형별 성공기준을 일관되게 적용할 수 있는 알고리즘 개발의 필요성이 제기된다.

본 연구에서는 MARS-KS 열수력 해석 코드의 시뮬레이션 출력을 대상으로, 다수의 사고유형에 걸쳐 일관되게 적용 가능한 자동 후처리 알고리즘 프레임워크를 개발하였다. 제안하는 프레임워크의 핵심은 절대 임계값에 의존하지 않는 상대 비교 기반 PRHRS 판정 알고리즘으로, 시뮬레이션 초기 조건이나 계통 규모가 변화하더라도 강인한 판정이 가능하도록 설계되었다. 또한 변수명 파일 기반 자동 매핑 기법을 도입하여 분석 대상 노형이 변경되는 경우에도 코드 수정 없이 적용할 수 있는 확장성을 확보하였다. 개발된 프레임워크는 급수상실사고(LOFW), 소형냉각재상실사고(SBLOCA), 일반과도(GTRN), 대형 2차측 파단(LSSB), 증기발생기 전열관 파단(SGTR)의 5개 사고유형에 적용되었으며, 각 사고유형의 물리적 특성에 근거한 파라미터 설정 기준을 제시한다. 본 연구에서 개발된 알고리즘은 향후 대용량 시나리오 데이터셋에 적용하여 성공기준 파라미터의 정량적 민감도 분석을 수행하는 후속 연구의 기반으로 활용될 것이다.

본 논문의 구성은 다음과 같다. 2장에서는 분석 대상 사고유형과 적용된 안전 기능 판정 기준을 설명하고, 3장에서는 개발된 알고리즘의 상세 설계를 기술한다. 4장에서는 개발된 알고리즘을 사고유형별 시뮬레이션 결과에 적용한 검증 결과를 제시하며, 5장에서 결론을 맺는다.

### 3.3 알고리즘 (핵심 섹션)

#### 3.3.1 변수명 자동 매핑 (VarMapper)
- 변수명 파일(MARS-KS 출력 기준) 파싱 → 역할별 컬럼 자동 추출
- 키워드 매칭 방식 (PRHRS_HX, PCT, RCP_HEAD)
- 노형 변경 시 확장 용이성

#### 3.3.2 원자로 트립(RT) 감지 알고리즘
- rktpow 80% 감소 시점 자동 추출 (초기값의 20% 이하)
- 데이터가 RT 이후 시작 시 fallback 처리 (time.min() 반환)
- PCT 계산용 RT (90% 감소 기준)와의 구분

#### 3.3.3 PRHRS 계통수 자동 판정 알고리즘 ← 논문 핵심
- **기존 절대 임계값 방식의 한계**: 시뮬레이션 조건 변화에 취약
- **상대 비교 알고리즘 제안**: `val ≥ max_val × RATIO`
- 알고리즘 파라미터 (사고유형별 물리적 근거 포함):

| 파라미터 | LOFW | SBLOCA | GTRN | LSSB | SGTR |
|---------|------|--------|------|------|------|
| 집계방식 | 전체평균 | 전체평균 | 전체평균 | 후반30% | 후반30% |
| WAIT (s) | 50 | 100 | 100 | 100 | 100 |
| FLOOR (W) | 1×10⁵ | 1×10⁵ | 1×10⁵ | 1×10⁵ | 1×10⁵ |
| 보정값 | 0 | 0 | 0 | 0 | 0 |

- **LSSB 후반 30% 평균 근거**: 초기 대형 과도현상에서 높은 열출력 발생,
  FLOOR 8×10⁵ W 적용으로 오판 방지
- **SGTR 파단 유로 보정 근거**: 파단 HX가 항상 최대값 → 실제 냉각 계통수
  과대계상 문제 보정 (CORRECTION = -1)

**인용**: [9] Di Maio et al. 2021, [10][11][12] PRHRS 신뢰도 논문

#### 3.3.4 PCT 최대값 추출 알고리즘
- RT 이후 구간 필터링 (rktpow 90% 감소 시점 기준)
- 이상값 제거 (> 10,000 K)
- cladding temp 키워드 기반 자동 컬럼 추출

**인용**: [2] 10 CFR 50.46, [3] Wilks 1941, [4] Martin & O'Dell 2005

### 3.4 구현 및 검증

#### 3.4.1 객체지향 설계
- BaseAnalyzer + 사고유형별 서브클래스 (LOFWAnalyzer, SBLOCAAnalyzer 등)
- 공통 로직 상속, 사고유형별 파라미터만 오버라이드

#### 3.4.2 RT 감지 알고리즘 검증
- 수작업 확인 결과와 알고리즘 결과 비교표
- 사고유형별 RT 시점 일치 여부

#### 3.4.3 PRHRS 계통수 판정 검증
- 수작업 확인 결과와 알고리즘 결과 비교표
- 사고유형별 판정 결과

#### 3.4.4 한계사항 및 적용 범위
- 검증 데이터셋이 OK 시나리오로만 구성됨
- PCT 임계값 기반 CD 판정의 직접 검증은 향후 과제
- ADS/PSIS/SIT: 현재 데이터셋에서 작동 임계값 미달 → N/A 처리
