## 1. 검토 대상

## 2. 라운드 1 — 개정 전 데이터

### 2.1 핵심 발견 지표

| 항목                                | 결과                                |
|-------------------------------------|-------------------------------------|
| PCT↔Outcome 일관성 (A)              | 5파일 모두 0건 위반 ✓               |
| Feed-and-Bleed 구조 (B)             | 5파일 모두 0건 위반 ✓               |
| PRHRS=4 → OK (C2)                   | 5파일 모두 100% ✓                   |
| PCT~PRHRS 상관 (C4)                 | -0.62 ~ -0.65 (음의 상관)           |
| **완전실패 → CD (C1)**              | **80~93%** (기대 ~100%)             |
| **ATWS → CD (C3)**                  | **LOFW 0%, SGTR 0%, LSSB 33%**      |

### 2.2 리뷰어 1차 의견 요지

**판정: Major Revision, Desk Reject 가능성 상당**

- ATWS 에서 LOFW/SGTR CD=0% 는 PSA 커뮤니티에서 **즉각적인 red flag**, 한 줄로 reject 당할 수 있음
- 현 검증은 알고리즘의 *실행 가능성(executability)* 만 보일 뿐, 변환된 Binary ET 의 PSA 의사결정 유의미성은 전혀 검증 못함
- 저자가 "시연용이다" 변명해도 **부분 수용**에 그침 — 시연용이라도 물리법칙 위배는 불허
- 최소 수정 요구:
  1. 최소 1개 사고유형에 대한 MARS-KS/RELAP5/MELCOR 실제 시뮬레이션 결과 추가
  2. Abstract/Conclusion 의 "verified/validated" 표현 전면 삭제
  3. ATWS / 완전실패 케이스의 물리적 비일관성을 Limitations 에 명시하거나 데이터 재생성
  4. 성공기준 탐색 (최적화/보수적) 민감도 분석 및 모드 간 Binary ET 구조 차이 정량화

### 2.3 엔지니어 1차 의견 요지

**첫 반응**: *"구조 로직은 잘 짰는데, 완전실패·ATWS 부분 튜닝이 덜 됐네"*

- ATWS CCDP 는 문헌상 **10⁻¹ ~ 10⁻²** (100% 는 아님). MTC / AMSAC / 수동 붕산주입 완화 경로 존재
- 그러나 **LOFW+ATWS 0%, SGTR+ATWS 0% 는 비현실**. 열제거원 상실 + 출력 미정지 → PCT 급등 거의 확정, 최소 **60~80% CD**
- PRHRS 4계통 → 100% OK 는 피동설계 특성상 수용 가능
- 완전실패 7~20% 생존은 **안 됨**. SIT 수십 m³ 재고로는 붕괴열 제거 불가. **CD ≥ 98%** 가 맞음
- 경계 PCT 분산 (1477K 근방 45~64% CD) 은 **오히려 현실적**. 실제는 PCT + 산화량(17% ECR) + 수소발생 복합 기준이어야 함
- **시연 목적 한정** 이라면 허용 가능, 단 "synthetic, 방법론 검증용" 명시 필수. SMART100 안전성 주장에 **절대** 인용 금지
- 우선 수정 3가지:
  1. RT=Fail + 열제거원 상실 → CCDP ≥ 0.7 강제
  2. PRHRS=0 & ADS=0 & PSIS=Fail → CD 100% 룰 고정
  3. 판정 기준 이중화 (PCT + 노심노출시간 / 산화량 proxy)

## 3. 수정 내용 — `demo_dataset_generator.py` 개정

엔지니어 권고 ①②를 `_sample_pct()` 에 2개 if-block 으로 최소 침습적 구현 (③은 향후 과제로 보류).

```python
# [수정 1] ATWS 보정 — RT=Fail 시 완화계통 상태별 차등 PCT 상향
if rt == 'Fail':
    if prhrs_ok:              # PRHRS ≥ 2계통 작동
        pct += uniform(400, 550)
    else:                      # PRHRS 작동 불충분
        pct += uniform(300, 500)

# [수정 2] 완전실패 하한 — SIT 단독 생존 불가
if (not prhrs_ok) and (not ads_ok) and (not psis_ok):
    pct = max(pct, uniform(1550, 1800))
```

## 4. 라운드 2 — 개정 후 데이터

### 4.1 개선 지표

| 항목                    | 개정 전     | 개정 후     |
|-------------------------|-------------|-------------|
| 완전실패 → CD (전 5파일) | 80~93%      | **5/5 = 100%** |
| ATWS CD — GTRN (n=3)     | 100%        | 100%        |
| ATWS CD — LOFW (n=4)     | **0%**      | **100%**    |
| ATWS CD — LSSB (n=3)     | 33%         | 67%         |
| ATWS CD — SBLOCA (n=10)  | 44%         | **100%**    |
| ATWS CD — SGTR (n=13)    | **0%**      | **77%**     |
| A/B 구조 규칙 위반      | 0건         | 0건 (유지)  |
| CD/OK 전체 분포         | 36~42% CD   | 38~44% CD   |

### 4.2 리뷰어 2차 의견 요지

**판정: Accept with Minor Revision (조건부)**

- 1차 라운드의 두 물리 위반 **명확히 해소**. 최소 침습적 구현을 긍정 평가
- 잔존 우려:
  1. **LSSB ATWS 67% (n=3)** — 표본 극소, 통계 해석 불가. "n<10 예시적" 주석 + 미달 1건의 물리 시나리오 각주
  2. **SBLOCA PRHRS=4 → OK 93.8%** — 6.2% CD 가 ATWS 중첩 1건. 조건부 분해 표(|ATWS, |non-ATWS) 제시 필요
  3. **경계구간 1403~1551K CD 27~62%** — 재포지셔닝 OK, Limitations 에 다변수 성공기준 확장 구체 후속과제 명시
- 합성 데이터 방법론 논문의 한계:
  - MARS-KS 비선형성/불확실성 전파 검증 부재
  - 실제 DET 가지치기 복잡도 하의 성공기준 탐색 수렴성 미확인
  - AIMS-PSA 출력 `.ket` 의 end-to-end 정량 결과와 기존 수동 ET 의 교차 검증 없음
  - Title/Abstract 에 "synthetic demonstration" 명시, "validation" 용어 제거 필수
- 저널 권고:
  - **RESS**: 현 수준 desk-reject 위험
  - **NED**: 경계선, MARS-KS 1개 사고유형 파일럿 포함 시 가능
  - **Annals of Nuclear Energy**: 현 원고에 가장 적합
- 추가 요구: (i) 규칙 추가 전후 민감도 분석 그림 1개, (ii) MARS-KS 소규모 파일럿 Section 포함 시 NED 급 재투고

### 4.3 엔지니어 2차 의견 요지

**판정: 조건부 통과 ("써도 된다")**

- 규칙 1 (ATWS 보정): 도식적이지만 시연용 충분. PRHRS 작동 시 PCT 피크 1400~1600K 는 iPOWER/SMART ATWS MARS-KS 결과와 근사 일치
- **다만 PRHRS 실패 측 +300~500K 은 약한 편** — 실제로는 +500~800K 까지 튐. 과보수가 아니라 "보수성이 덜함"
- 규칙 2 (완전실패 floor 1550~1800K): **적정**. SBLOCA + PRHRS/ADS/PSIS 전부 실패는 30분~1시간 내 노심노출 → 1800K 돌파. 1550K 하한은 약간 관대하나 수용
- SBLOCA/LOFW ATWS 100% 는 **과보수 아님** — 이항분포 신뢰구간상 n=10, n=4 에서 실제 모비율 75% 여도 100% 관측은 드물지 않음. **n≥30 으로 재확인 필요**
- LSSB 67%, SGTR 77% 는 통계·물리적으로 안정. **통과**
- 판정 이중화 (규칙 ③) 생략은 방법론 시연 논문에선 수용 가능. 단 Limitation 에 "실무 적용 시 PCT + 산화량(ECR 17%) + 노심노출시간 다중 기준 확장 필요" 한 문장 필수
- 최종: "써도 된다" — 단 (a) ATWS n≥30 안정성 재확인, (b) Limitation 에 ③ 명시

## 5. 서론 / Abstract / Limitations 작성 시 반영 사항

두 에이전트 수렴 사항을 서론 및 관련 섹션에 **반드시** 포함.

### 5.1 프레이밍 (Abstract / Introduction)

- ❌ **금지어**: "validate", "verify", "proven", "안전성 입증"
- ✅ **권장어**: "demonstrate", "illustrate", "methodology pipeline", "proof-of-concept"
- 데이터 성격을 첫 언급에서 **"synthetic dataset for pipeline illustration"** 로 명시
- 연구 기여를 "알고리즘 제안 + 파이프라인 시연" 으로 한정, "SMART100 안전성 평가" 로 확장 해석 금지

### 5.2 Limitations 섹션 필수 3문장

1. **단일 PCT 임계치 한계**: 현 판정은 PCT_max ≥ 1477K 단일 기준. 실무 적용 시 PCT + 산화량(ECR 17%) + 노심노출시간의 다중 기준 확장 필요.
2. **합성 데이터 한계**: 본 데모 데이터는 파이프라인 시연을 위한 synthetic 이며, MARS-KS / RELAP5 실제 열수력 시뮬레이션 결과로의 검증은 향후 과제.
3. **보수성 재검토**: ATWS 보정 규칙의 PRHRS 실패 측 PCT 가산 폭(+300~500K) 은 실제 해석 결과(+500~800K) 대비 다소 약하며, 실무 적용 전 재보정 필요.

### 5.3 Introduction 연구 기여 진술 (Suggested)

```
본 연구는 대규모 DET 시뮬레이션 결과를 PSA 표준 Binary ET 로 자동 변환하는
후처리 알고리즘을 제안하고, AIMS-PSA 호환 .ket 출력까지의 end-to-end
파이프라인을 시연한다. 데이터 파이프라인의 작동 원리와 출력 산출물의 형식
호환성을 입증하기 위해 합성 데이터셋(5개 사고유형 × 200 시나리오)을 사용하며,
실제 열수력 시뮬레이션 결과와의 검증은 후속 연구의 범위로 한다.
```

### 5.4 서론에서 선제적으로 답변해야 할 리뷰어 질문

- Q1. "왜 실제 MARS-KS 결과를 안 쓰고 합성 데이터를 쓰는가?"
  → A1. 본 연구의 기여는 알고리즘·파이프라인 자체이며, 물리 엔진 검증은 별도 연구. 합성 데이터로도 알고리즘의 입출력 형식 호환성 / 성공기준 탐색 수렴성 / Binary 변환 논리의 정합성은 충분히 시연 가능하다.
- Q2. "합성 데이터의 물리적 타당성은?"
  → A2. 데이터 생성기는 SMART100 설계 문서 및 PSA 관례(Feed-and-Bleed, ATWS 완화 경로)를 반영하며, 1000건 전수검사로 구조 논리 위반 0건을 확인 (Appendix 포함).
- Q3. "Binary ET 의 실제 유용성은 어떻게 보이나?"
  → A3. 변환된 `.ket` 을 AIMS-PSA 에서 재렌더링하여 시각적/구조적 일관성을 보인다 (Figure X). 정량적 PSA 지표 (Minimal Cut Set, 중요도) 재현성은 실제 데이터 연동 후속 과제.
