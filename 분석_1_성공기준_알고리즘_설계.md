## 1. 알고리즘 설계 원칙

### 1.1 왜 이 알고리즘이 필요한가

현재 `et_generator.py`는 `PRHRS_count` 컬럼의 값(0, 1, 2, 3, 4)을 그대로 분기점으로 사용한다. 이는 DET(Dynamic Event Tree) 방식의 출력을 그대로 반영한 것으로, PSA에서 요구하는 Binary ET 구조와 근본적으로 다르다.

Binary ET에서 각 안전계통의 헤딩은 반드시 **성공(Success) / 실패(Failure)** 두 가지로만 분기해야 한다. 이를 위해서는 "몇 개 계통이 작동해야 성공인가"라는 **성공기준(Success Criteria)**이 먼저 정의되어야 한다.

성공기준은 다음 이유로 미리 알기 어렵다:
- SMART100 PRHRS는 4개의 수동냉각계통으로 구성되며, 2/4, 3/4 등 다양한 N/M 조합이 가능하다.
- 성공기준은 사고 시나리오의 열수력 조건(앞 계통의 성공 여부 등)에 따라 달라진다.
- 수동계통이므로 실험/규제 데이터가 제한적이며, 시뮬레이션 결과로부터 역산하는 것이 현실적이다.

따라서, MARS-KS 시뮬레이션 결과(CSV)에서 PCT 응답을 기반으로 **성공기준을 데이터 주도적으로 자동 탐색**하는 알고리즘이 필요하다.

### 1.2 PSA 도메인에서의 성공기준 정의

```
성공기준: "M개의 계통 중 N개 이상이 작동하면 노심을 보호할 수 있다"
표기법: N/M  (예: 2/4 = 4개 중 2개 이상 작동 시 성공)
```

노심 보호 판단 기준: NRC 10 CFR 50.46 기준 **PCT < 1477 K** (1204°C, ~2200°F)

## 2. 입력 데이터 구조

### 2.1 CSV 컬럼 정의

| 컬럼명 | 자료형 | 설명 | 예시 값 |
|--------|--------|------|---------|
| `Scenario` | str | 시나리오 파일명 | `SGTR_22` |
| `Reactor_Trip` | str | 원자로 급정지 성공 여부 | `Success` / `Fail` |
| `RCP_Status` | str | 냉각재 펌프 상태 | `Running` / `Coast-down` / `Natural Circulation` |
| `PRHRS_count` | int | PRHRS 작동 계통수 (0~4) | `0`, `1`, `2`, `3`, `4` |
| `PCT_max` | float | 시뮬레이션 최고 피복재 온도 (K) | `862.89` |
| `Outcome` | str | 노심손상 여부 | `OK` / `CD` |

### 2.2 알고리즘이 직접 사용하는 컬럼

- **조건 변수**: `Reactor_Trip`, `RCP_Status` (앞 계통 상태)
- **대상 계통 변수**: `PRHRS_count` (0~4)
- **결과 변수**: `PCT_max` (연속값), `Outcome` (검증용)

## 3. 최적화 모드 알고리즘 상세 설계

### 3.1 PCT 20% 마진 기준 설명

```
규제 한계치:     PCT_limit     = 1477 K
안전 여유도:     margin        = 20%
마진 적용 기준: PCT_threshold = 1477 × (1 - 0.20) = 1181.6 K
```

20% 마진의 공학적 근거:
- 시뮬레이션 모델 불확실성(코드 편향, 입력 불확실성) 흡수
- DET 방식 특성상 특정 노드 조합의 샘플 수가 적을 경우 보수성 확보
- 규제 기관에 대한 수용 가능한 안전 여유도 제시

> 마진은 파라미터로 조정 가능하며, 0%로 설정하면 1477 K를 직접 기준으로 사용.

### 3.2 브랜치별 순회 방법

```
전체 분기 경로 = {앞 계통 상태 조합}
예 (Reactor_Trip 조건):
  - 경로 1: Reactor_Trip=Success
  - 경로 2: Reactor_Trip=Fail
```

각 경로에 해당하는 시나리오 행을 필터링하고, 그 부분집합에서 PRHRS 성공기준을 독립적으로 탐색한다.

### 3.3 성공/실패 경계 탐색 로직

PRHRS_count는 0~4의 이산 정수이므로 **순차 탐색(Sequential Search)**을 사용한다.

```
탐색 방향: 높은 계통수(4) → 낮은 계통수(0)
이유: 가장 완화된(least restrictive) 성공기준을 찾는 최적화에 해당
```

PCT 대표값 선택 정책:

| 정책 | 특성 |
|------|------|
| 평균 mean | 완화적, 통계적 대표성 |
| 95퍼센타일 p95 | 준보수적, **PSA 기본값** |
| 최대값 max | 극보수적, 단일 최악 케이스 |

### 3.4 앞 계통 상태에 따른 조건부 성공기준 처리

각 브랜치 경로를 독립적으로 탐색하여 경로별 성공기준을 별도 저장한다.

```
예시 결과:
  Reactor_Trip=Success → PRHRS 성공기준: 2/4
  Reactor_Trip=Fail    → PRHRS 성공기준: 3/4
```

## 4. 보수적 모드 알고리즘 상세 설계

### 4.1 N/M 후보 생성 방법

```
M=4 기준 후보:
  1/4 (1개 이상 작동)
  2/4 (2개 이상 작동)
  3/4 (3개 이상 작동)
  4/4 (4개 모두 작동)
```

각 N/M 후보에 대해 `PRHRS_count >= N` 조건의 PCT_max 최대값을 계산한다.

### 4.2 더 엄격한 기준 자동 선택 로직

```
탐색 방향: 낮은 N(1) → 높은 N(4)
선택 기준: PCT < 1477 K 를 만족하면서 가장 높은 N값 선택
이유: 안전 측면에서 가장 보수적(엄격한) 성공기준
```

### 4.3 안전 여유도 처리

보수적 모드 기본값: 마진 없음 (N값을 가장 높게 선택하는 것 자체가 보수적). 추가 마진 적용 시 이중 보수성 발생 가능 → 선택적 파라미터로 운영.

## 5. 앞 계통 → 뒤 계통 의존성 처리 방법

### 5.1 조건부 성공기준 데이터 구조 설계

```python
{
    "system": "PRHRS",
    "total": 4,
    "mode": "optimization",
    "margin": 0.20,
    "pct_threshold": 1181.6,
    "conditions": [
        {
            "condition_vars": {"Reactor_Trip": "Success"},
            "n_scenarios": 12,
            "min_count": 2,
            "criteria_str": "2/4",
            "pct_at_criteria": {"mean": 750.3, "p95": 820.1, "max": 862.9},
            "confidence": "HIGH"   # n>=10:HIGH / n>=5:MEDIUM / n>=3:LOW / n<3:INSUFFICIENT
        },
        {
            "condition_vars": {"Reactor_Trip": "Fail"},
            "n_scenarios": 10,
            "min_count": 3,
            "criteria_str": "3/4",
            "pct_at_criteria": {"mean": 890.2, "p95": 950.0, "max": 1020.5},
            "confidence": "MEDIUM"
        }
    ]
}
```

### 5.2 계통 간 상관관계 트리 구성

```
Level 0 (IE):  개시사건
Level 1:       Reactor_Trip (1차 안전계통)
Level 2:       PRHRS        (2차 안전계통) ← Level 1 상태에 조건부
Level 3:       ADS / PSIS   (추가 계통)    ← Level 1,2 상태에 조건부
```

처리 원칙: 뒤 계통 탐색 시 앞 계통 상태를 모두 조건으로 포함. PRHRS 바이너리 변환이 선행되어야 ADS 조건부 탐색이 가능하므로 **계층 순서대로 순차 처리**.

### 6.1 최종 출력: 성공기준 결과 객체


## 7. 구현 시 주의사항 및 엣지케이스

### 7.1 샘플 부족 문제
- n < 3: 탐색 불가 → 글로벌 기준으로 fallback
- n < 5: LOW 신뢰도 경고 출력 후 탐색 진행
- n = 0: 물리적으로 불가능한 조합으로 기록

### 7.2 단조성(Monotonicity) 위반
계통수 증가 시 PCT 단조 감소 원칙이 샘플 변동으로 역전될 수 있음. 대응: 누적 최소값(cumulative minimum) 보정 적용.

```python
# 예: {0: 1200, 1: 950, 2: 800, 3: 820(역전), 4: 700}
# 보정: {0: 1200, 1: 950, 2: 800, 3: 800, 4: 700}
```

### 7.3 모든 계통수에서 CD 발생
PRHRS만으로 노심 보호 불가. 성공기준 None 반환, 해당 브랜치 전체 CD로 종결 처리.

### 7.4 모든 계통수에서 OK (0계통 포함)
PRHRS가 이 브랜치에서 불필요함. 성공기준 0/4 반환, 경고 출력.

### 7.5 PRHRS_count 음수 케이스
~~SGTR에서 `PRHRS_CORRECTION=-1` 보정으로 음수 발생 가능.~~ → **해결 완료**: `psa_analyzer.py`에서 `max(0, count + PRHRS_CORRECTION)`으로 CSV 출력 시점에 음수를 차단함 (line 340). `success_criteria.py`의 `clip(0, M)`은 방어적 코드로 유지.

### 7.6 PCT 이상치
탐색 전 PCT < 10000 K 필터 유지. 추가로 IQR 기반 이상치 감지 및 플래그 처리 권장.

### 7.7 조건 변수 카테고리 수 폭발
앞 계통 3개 × 상태 3개 = 27 조합 → 각 조합당 샘플 부족 가능. 대응: 임계값(기본 5개) 미만 조합은 상위 레벨로 그룹핑.

## 8. Python 의사코드 (Pseudo-code)

### 8.1 메인 클래스

```python
class SuccessCriteriaFinder:
    """
    성공기준 자동 탐색 클래스

    Parameters
    ----------
    df                : pd.DataFrame  psa_analyzer 출력 CSV
    target_system     : str           'PRHRS_count'
    system_capacity   : int           전체 계통 수 (SMART100: 4)
    condition_vars    : list[str]     앞 계통 컬럼명 목록 ['Reactor_Trip', ...]
    pct_col           : str           'PCT_max'
    pct_limit         : float         1477.0 K (NRC 규제 기준)
    margin            : float         0.20 (20% 마진, 최적화 모드)
    mode              : str           'optimization' | 'conservative'
    pct_representative: str           'p95' | 'mean' | 'max'
    min_samples       : int           최소 샘플 수 (기본: 5)
    """

    PCT_LIMIT_DEFAULT = 1477.0

    def __init__(self, df, target_system='PRHRS_count', system_capacity=4,
                 condition_vars=None, pct_col='PCT_max',
                 pct_limit=1477.0, margin=0.20,
                 mode='optimization', pct_representative='p95',
                 min_samples=5):
        self.df                 = df.copy()
        self.target_col         = target_system
        self.M                  = system_capacity
        self.condition_vars     = condition_vars or []
        self.pct_col            = pct_col
        self.pct_limit          = pct_limit
        self.margin             = margin
        self.mode               = mode
        self.pct_representative = pct_representative
        self.min_samples        = min_samples

        # PCT 임계값 계산
        if mode == 'optimization':
            self.pct_threshold = pct_limit * (1.0 - margin)
        else:
            self.pct_threshold = pct_limit  # 보수적 모드: 마진 없음

        # 이상치 제거 및 유효 범위 제한
        self.df = self.df[self.df[pct_col] < 10000].copy()
        self.df[target_system] = (
            self.df[target_system].clip(lower=0, upper=system_capacity).astype(int)
        )

    def run(self):
        """성공기준 탐색 실행 → SuccessCriteriaResult 반환"""
        # 1. 글로벌 성공기준 탐색 (조건 없이 전체 데이터)
        global_result = self._find_criteria_for_subset(self.df, label="GLOBAL")

        # 2. 조건부 성공기준 탐색 (앞 계통 상태별)
        conditional_results = []
        if self.condition_vars:
            for condition_dict in self._enumerate_conditions():
                subset = self._filter_by_condition(self.df, condition_dict)
                if len(subset) < self.min_samples:
                    result = self._make_fallback(condition_dict, global_result)
                else:
                    result = self._find_criteria_for_subset(
                        subset, label=str(condition_dict), condition_vars=condition_dict
                    )
                conditional_results.append(result)

        # 3. 이진 매핑 및 컬럼 추가
        binary_mapping = self._build_binary_mapping(conditional_results, global_result)
        self.df['PRHRS_binary'] = self._apply_binary_mapping(binary_mapping)

        return self._assemble_result(global_result, conditional_results, binary_mapping)

    def _enumerate_conditions(self):
        """condition_vars의 모든 카테고리 조합 열거"""
        from itertools import product
        unique_values = {
            var: sorted(self.df[var].dropna().unique())
            for var in self.condition_vars if var in self.df.columns
        }
        vars_list = list(unique_values.keys())
        for combo in product(*[unique_values[v] for v in vars_list]):
            yield dict(zip(vars_list, combo))

    def _filter_by_condition(self, df, condition_dict):
        mask = pd.Series([True] * len(df), index=df.index)
        for var, val in condition_dict.items():
            if var in df.columns:
                mask &= (df[var] == val)
        return df[mask].copy()

    def _find_criteria_for_subset(self, subset, label="", condition_vars=None):
        if self.mode == 'optimization':
            return self._find_criteria_optimization(subset, label, condition_vars)
        else:
            return self._find_criteria_conservative(subset, label, condition_vars)

    def _find_criteria_optimization(self, subset, label, condition_vars):
        """최적화 모드: 높은 계통수부터 순회, PCT < threshold 인 최소 N 탐색"""
        pct_stats = self._compute_pct_stats_by_count(subset)
        pct_stats = self._enforce_monotonicity(pct_stats)

        min_count = None
        for count in range(self.M, -1, -1):  # 4, 3, 2, 1, 0
            if count not in pct_stats:
                continue
            stats = pct_stats[count]
            if stats['n'] < self.min_samples:
                continue
            if stats[self.pct_representative] < self.pct_threshold:
                min_count = count  # 더 낮은 계통수도 확인

        if min_count is None:
            criteria_str, final_count = "None (노심 보호 불가)", None
        elif min_count == 0:
            criteria_str, final_count = "0/4 (PRHRS 불필요)", 0
        else:
            criteria_str, final_count = f"{min_count}/{self.M}", min_count

        return self._build_criteria_record(
            condition_vars, subset, final_count, criteria_str, pct_stats, label
        )

    def _find_criteria_conservative(self, subset, label, condition_vars):
        """보수적 모드: 낮은 계통수부터 순회, PCT < threshold 만족하는 가장 높은 N 선택"""
        best_count = None
        for count in range(1, self.M + 1):  # 1, 2, 3, 4
            subset_ge = subset[subset[self.target_col] >= count]
            if len(subset_ge) == 0:
                break
            if subset_ge[self.pct_col].max() < self.pct_threshold:
                best_count = count  # 더 엄격한 N+1도 확인

        criteria_str = (f"{best_count}/{self.M}" if best_count
                        else "None (어떤 N/M도 안전 보장 불가)")
        pct_stats = self._compute_pct_stats_by_count(subset)
        return self._build_criteria_record(
            condition_vars, subset, best_count, criteria_str, pct_stats, label
        )

    def _compute_pct_stats_by_count(self, subset):
        """계통수별 PCT 통계 (n, mean, p95, max, min) 집계"""
        stats = {}
        for count in range(0, self.M + 1):
            rows = subset[subset[self.target_col] == count]
            if len(rows) == 0:
                continue
            pct_vals = rows[self.pct_col].dropna()
            stats[count] = {
                'n':    len(pct_vals),
                'mean': float(pct_vals.mean()),
                'p95':  float(pct_vals.quantile(0.95)) if len(pct_vals) >= 2 else float(pct_vals.max()),
                'max':  float(pct_vals.max()),
                'min':  float(pct_vals.min()),
            }
        return stats

    def _enforce_monotonicity(self, pct_stats):
        """단조성 보정: 계통수 증가 → PCT 단조 감소 원칙 강제 적용"""
        min_so_far = float('inf')
        for count in sorted(pct_stats.keys()):  # 낮은 count → 높은 count 순회
            cur = pct_stats[count][self.pct_representative]
            if cur > min_so_far:
                pct_stats[count][self.pct_representative] = min_so_far
                pct_stats[count]['monotonicity_corrected'] = True
            else:
                min_so_far = cur
                pct_stats[count]['monotonicity_corrected'] = False
        return pct_stats

    def _build_criteria_record(self, condition_vars, subset, min_count,
                                criteria_str, pct_stats, label):
        n = len(subset)
        confidence = ("HIGH" if n >= 10 else "MEDIUM" if n >= 5
                      else "LOW" if n >= 3 else "INSUFFICIENT")
        return {
            "label":           label,
            "condition_vars":  condition_vars or {},
            "n_scenarios":     n,
            "min_count":       min_count,
            "criteria_str":    criteria_str,
            "pct_at_criteria": pct_stats.get(min_count, {}) if min_count is not None else {},
            "confidence":      confidence,
            "pct_stats_all":   pct_stats,
        }

    def _make_fallback(self, condition_dict, global_result):
        return {
            "label": str(condition_dict), "condition_vars": condition_dict,
            "n_scenarios": 0, "min_count": global_result['min_count'],
            "criteria_str": global_result['criteria_str'] + " [FALLBACK]",
            "pct_at_criteria": {}, "confidence": "INSUFFICIENT", "pct_stats_all": {},
        }

    def _build_binary_mapping(self, conditional_results, global_result):
        """조건 튜플 → N값 매핑. 예: {("Success",): 2, ("Fail",): 3}"""
        if not conditional_results:
            return {tuple(): global_result['min_count']}
        return {
            tuple(r['condition_vars'].values()): r['min_count']
            for r in conditional_results
        }

    def _apply_binary_mapping(self, binary_mapping):
        """PRHRS_binary 컬럼 생성 (Success / Fail)"""
        def classify_row(row):
            key = tuple(row[v] for v in self.condition_vars) if self.condition_vars else tuple()
            n_threshold = binary_mapping.get(key)
            if n_threshold is None:
                return "Unknown"
            if n_threshold == 0:
                return "Success"
            return "Success" if row[self.target_col] >= n_threshold else "Fail"
        return self.df.apply(classify_row, axis=1)

    def _assemble_result(self, global_result, conditional_results, binary_mapping):
        return {
            "metadata": {
                "target_system": self.target_col, "system_capacity": self.M,
                "mode": self.mode, "pct_limit_K": self.pct_limit,
                "margin_pct": self.margin * 100, "pct_threshold_K": self.pct_threshold,
                "n_total": len(self.df),
            },
            "global_criteria":      global_result,
            "conditional_criteria": conditional_results,
            "binary_mapping":       binary_mapping,
            "df_with_binary":       self.df,
        }
```

## 참고 사항

### 관련 파일

| 파일 | 역할 |
|------|------|
| `psa_analyzer.py` | MARS-KS → CSV (PRHRS_count, PCT_max 포함) |
| `et_generator.py` | CSV → ET Excel (현재 DET 구조) |
| `sgtr_results.csv` | SGTR 시나리오 결과 예시 데이터 |
| `success_criteria.py` | (신규 개발 예정) 본 문서의 알고리즘 구현 |

### 구현 우선순위

1. `SuccessCriteriaFinder` 클래스 기본 구현 (단일 조건 변수)
2. 데모용 데이터셋 생성 (앞 계통 의존성이 반영된 합성 데이터)
3. 다중 조건 변수 및 계층적 의존성 처리
4. `et_generator.py`와의 연동 (이진 컬럼 전달)
5. 신뢰도 보고서 및 경고 메시지 시스템

### 알고리즘 검증 방법

1. **교차 검증**: 탐색된 N을 원본 데이터에 적용하여 CD 분류 정확도 확인
2. **물리적 타당성 검토**: 기존 문헌 성공기준과 비교
3. **민감도 분석**: 마진(0%, 10%, 20%, 30%)에 따른 성공기준 변화
4. **샘플 수 영향 분석**: 최소 샘플 수 기준에 따른 결과 안정성 평가

## 핵심 인사이트 (현재 코드 분석 결과)

- `psa_analyzer.py`의 `CD_THRESHOLD = 1477` (BaseAnalyzer)이 규제 한계치로 이미 정의되어 있음
- `et_generator.py`는 `PRHRS_count` 값을 정수 그대로 분기 처리 중 (DET 구조)
- 현재 `sgtr_results.csv`는 모든 PCT_max가 1477K 미만(최대 862.89K)이어서 CD가 없음
  → **성공기준 탐색 테스트를 위한 합성 데이터셋이 필수**

**두 모드의 핵심 차이 요약:**

| 항목 | 최적화 모드 | 보수적 모드 |
|------|------------|------------|
| PCT 임계값 | 1477 × 0.8 = 1182 K | 1477 K (마진 없음) |
| 탐색 방향 | 높은 N → 낮은 N | 낮은 N → 높은 N |
| 선택 기준 | 가장 완화된(최소) N | 가장 엄격한(최대) N |

**et_generator.py 연동 시 최소 수정 포인트:**
`PRHRS_count` 헤딩을 `PRHRS_binary`로 교체하면 기존 `_build_branches()` 및 `_draw_tree()` 로직을 그대로 활용할 수 있음.
