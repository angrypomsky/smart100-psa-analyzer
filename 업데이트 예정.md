# 업데이트 예정 기능 목록

> 작성일: 2026-03-25 | 원본: `성공기준에 맞춰 binary하게 et구조 생성하는 기능.txt`

---

## 핵심 목표

**PSA 후처리 자동화** — 대규모 시뮬레이션 결과를 자동 가공하여
기존 ET(이벤트 트리) 형태로 변환하는 모듈 개발

---

## 1. ET 자동 생성 기능 ★★★

### 1.1 기본 흐름
```
Excel 결과 (psa_analyzer 출력)
        ↓  [Confirm 클릭]
KET 구조 변환
        ↓
AIMS ET 구조 생성
```

### 1.2 Binary ET 구조
- 각 안전계통의 성공/실패를 Binary하게 분기
- 성공기준에 따라 브랜치 자동 결정
- 앞 계통의 성공/실패 여부가 뒤 계통의 성공기준에 영향

**예시:**
```
앞 계통 성공 → 뒤 계통 성공기준: 2/4
앞 계통 실패 → 뒤 계통 성공기준: 3/4
```

---

## 2. 성공기준 자동 탐색 알고리즘 ★★★

두 가지 모드 중 선택하여 실행:

### 2.1 최적화 모드 (Optimization)
- PCT 20% 마진 기준으로 성공/실패 경계 자동 탐색
- 자동으로 브랜치를 순회하며 성공기준 확인
- 반복 최적화를 통해 최적 성공기준 도출

### 2.2 보수적 모드 (Conservative)
- 안전계통 N/M 구성에서 더 엄격한 기준 자동 선택
  - 예: 2/4와 3/4 중 → 3/4 선택
- 안전 여유도를 보수적으로 유지

> **구현 방향**: 실행 시 최적화/보수적 중 선택 → 해당 알고리즘으로 ET 자동 생성

---

## 3. 데모용 데이터셋 생성 ★★★ `완료`

### 3.1 요구사항
- 기존 통념(경수로 물리적 특성)에 맞는 그럴싸한 데이터
- 앞 계통 성공/실패에 따라 뒤 계통 성공기준이 다르게 나오는 구조
- 기존 경수로 데이터를 데모 목적에 맞도록 수정하여 사용

### 3.2 데이터셋 구성 조건
- PRHRS 계통수 분포: 물리적으로 타당한 범위
- CD/OK 비율: 현실적인 PSA 결과와 유사하게
- 계통 간 상관관계 반영 (앞 계통 → 뒤 계통 영향)

### 3.3 구현 완료 사항 (2026-04-01)
- `demo_dataset_generator.py` 구현 완료
- 5개 사고유형 지원: LOFW, SBLOCA, GTRN, LSSB, SGTR
- 안전계통 순차 샘플링: RT → RCP → PRHRS → ADS → PSIS → SIT → PCT
- Feed-and-Bleed 로직: PRHRS≤1 → ADS 진입, ADS=0 → PSIS=Fail
- PCT 1477K 기준 CD/OK 판정
- ET 로직 검증 완료 (1000건 전수 위반 0건)

### 3.4 개선 검토 중
- **샘플링 방식 개선**: 현재 몬테카를로 방식은 OK 편향 심함 (CD 1.5~4.5%)
  - 계층화 샘플링(Stratified) 또는 LHS(Latin Hypercube) 방식 검토 중
  - ET 브랜치별 균등 분배 방식도 고려
- **대시보드 차트 변경 검토**: 히스토그램 → 산점도 변경 검토 중
  - 대안1: x=PCT, y=PRHRS (상관관계 시각화)
  - 대안2: 히스토그램 + 산점도 병행
  - 대안3: PCT 오름차순 정렬 계단형 곡선

## 6. 논문 서론 방향 (ET 생성 기능 관련)

> DET(Dynamic Event Tree) + 안전해석 시뮬레이션 → PSA 연계
> → 대규모 시뮬레이션 수행 후 인사이트 도출
> → **후처리 모듈 필요성**: 많은 데이터를 자동 가공하여 기존 ET 형태로 변환

- 관련 참고문헌 다수 인용 필요
- 기능요건 중심으로 작성
